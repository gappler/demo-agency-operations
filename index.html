Sidebar Creative is a 40-person digital agency in Portland doing $6.5M in annual revenue. Like many agencies that have grown fast, their operational infrastructure hasn't kept pace. Scope creep is endemic. Resource loading is invisible. Profitability is a black box until quarter-end. And every Friday afternoon, account directors spend 2-3 hours per client manually assembling performance reports in Google Slides.

This project demonstrates how Claude Code can build a complete suite of interconnected operational tools from a single brand definition and data set. Each tool was built in a single conversation using natural language prompts — no frameworks, no dependencies, no build steps. Every tool is a self-contained HTML file that opens in any browser.

Tools 1–3 cover project management — timelines, resources, and capacity. Tools 4–7 cover client reporting — status updates, performance trends, health scoring, and automated reports.

See the GitHub repo for brand, data, and other related context.

Each tool serves a unique purpose but they all share the same data and inform each other.

How This Was Built

The build order was the strategy. Before writing a single line of dashboard code, we created two foundation files: a brand definition that captured Sidebar Creative's identity, voice, and visual system, and a structured data set that modeled the entire business — 40 team members, 8 retainer clients, SOW terms, 6 months of performance history, and a project pipeline.

Every tool was then built sequentially, each one reading from the same foundation and inheriting context from the tools before it. The Timeline Dashboard established the project data model. The Resource Allocator layered team assignments on top of it. The Capacity Planner projected both forward. The client reporting tools consumed everything upstream — health scores informed the report generator, performance trends fed the dashboard, status reports pulled from timelines and budgets. No tool was built in isolation. The sequence was the architecture.

This matters because it's the opposite of how most internal tools get built. Typically someone builds a one-off spreadsheet, then another, then a dashboard that doesn't talk to either. Here, a single conversation with Claude Code produced each tool in minutes, and because every tool read from the same data and brand definition, they came out consistent — same colors, same client names, same numbers, same voice.
